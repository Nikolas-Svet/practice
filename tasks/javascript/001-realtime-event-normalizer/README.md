# Нормализация realtime-событий

## Цель

Отработать чистую JavaScript-логику для обработки потока событий: валидацию входных данных, нормализацию разных форматов и сбор итогового состояния без фреймворков.

Задача рассчитана примерно на 30 минут и опирается на опыт с frontend-интеграциями, WebSocket/SSE, REST API и unit-тестами.

## Требования

- Реализовать функцию `buildDeviceState(events)` в `src/index.js`.
- Функция принимает массив событий от условного realtime API.
- Каждое событие может иметь один из типов:
  - `connected`
  - `disconnected`
  - `position`
  - `battery`
- События без `deviceId`, без `timestamp` или с неизвестным `type` нужно игнорировать.
- Для каждого устройства нужно собрать итоговое состояние:
  - `deviceId`
  - `online`
  - `lastSeen`
  - `position`
  - `battery`
  - `warnings`
- Если событие `connected`, устройство становится `online: true`.
- Если событие `disconnected`, устройство становится `online: false`.
- Если событие `position`, нужно обновить `position` только если есть числовые `lat` и `lng`.
- Если событие `battery`, нужно обновить `battery` только если значение находится в диапазоне от `0` до `100`.
- Если батарея ниже `20`, добавить предупреждение `"low-battery"` в `warnings`.
- Предупреждение не должно дублироваться.
- Если для одного устройства приходят события в разном порядке, итоговое состояние должно учитывать только события с самым новым `timestamp` для полей `online`, `position` и `battery`.
- Результат должен быть массивом устройств, отсортированным по `deviceId` по алфавиту.

## Критерии приемки

- `buildDeviceState(events)` возвращает корректное итоговое состояние устройств.
- Невалидные события не ломают обработку и игнорируются.
- Старые события не перетирают более новые данные.
- Пограничные случаи обработаны: пустой массив, неизвестный тип события, некорректные координаты, батарея вне диапазона.
- Функция не мутирует исходный массив событий и сами объекты событий.

## Дополнительно

- Не использовать сторонние библиотеки.
- Не использовать DOM API.
- Стараться держать код читаемым: небольшие вспомогательные функции допустимы.
- Для самопроверки можно использовать сценарии из `tests/buildDeviceState.test.js`.
