# Правила практики

## Общие правила

- Делать каждое задание маленьким и сфокусированным.
- Предпочитать читаемый код хитрым решениям.
- Не смешивать несвязанные темы в одном задании.
- Добавлять заметки после завершения задания.

## Правила постановки задачи

- Для каждой задачи создается отдельная папка.
- В папке задачи обязательно должен быть `README.md` с описанием условия.
- В папке задачи обязательно должен быть `src/` с файлом решения.
- Основной файл решения должен называться понятно: `index.js`, `index.ts` или `App.vue`.
- Дополнительные файлы решения можно добавлять только внутри папки этой задачи.
- Тесты, если они нужны, хранятся в `tests/` внутри папки задачи.
- Заметки после выполнения хранятся в `notes.md` внутри папки задачи.

## Критерии завершения

- Требования задания реализованы.
- Пограничные случаи учтены.
- Тесты добавлены, если поведение не тривиальное.
- В `notes.md` есть короткое описание того, что было отработано.

## Стандарт коммитов

Формат сообщения:

```text
<тип>(<область>): <краткое описание>
```

- Писать сообщение на русском языке.
- Использовать прошедшее время от первого лица: `добавил`, `исправил`, `обновил`, `описал`.
- Делать один коммит на одну логически завершенную задачу.
- Область указывать по разделу или задаче: `javascript`, `typescript`, `vue`, `docs`, `templates`, `repo`.
- Если область не нужна, можно писать без нее: `<тип>: <краткое описание>`.

Типы коммитов:

- `feat` - новая задача, функциональность или учебный пример.
- `fix` - исправление ошибки в решении или тестах.
- `docs` - изменения документации, правил, заметок.
- `test` - добавление или обновление тестов.
- `refactor` - улучшение кода без изменения поведения.
- `chore` - служебные изменения структуры, настроек, шаблонов.

Примеры:

```text
feat(javascript): добавил задачу по нормализации realtime-событий
fix(javascript): исправил обработку нулевых координат
docs(repo): описал стандарт коммитов
```
