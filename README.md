# Frontend Practice

Создай мне задачу, учитывая мой опыт и знания, опираясь на мое резюме info.pdf. Задачу про javascript на полчаса. Все требования постановки задачи смотри в /templates



Репозиторий для регулярной практики фронтенда: JavaScript, TypeScript и Vue.

Цель - хранить маленькие задания в понятной структуре, повторять базовые навыки и публиковать готовые решения как историю практики.

## Структура

- `docs/` - заметки о процессе, правила и план развития
- `tasks/javascript/` - маленькие задания по JavaScript
- `tasks/typescript/` - маленькие задания по TypeScript
- `tasks/vue/` - маленькие задания по Vue
- `mini-projects/` - небольшие законченные приложения
- `playground/` - эксперименты и быстрые проверки
- `templates/` - шаблоны для новых заданий

## Формат задания

У каждого задания должна быть отдельная папка:

```text
tasks/<section>/<number>-<task-name>/
  README.md
  src/
    index.js | index.ts | App.vue
  tests/
  notes.md
```

## Правила постановки задачи

- Каждая задача создается в отдельной папке.
- Внутри папки обязательно должен быть `README.md` с описанием задачи.
- Внутри папки обязательно должен быть файл с решением в `src/`.
- Название папки должно начинаться с номера задачи: `001-task-name`.
- Если решение требует нескольких файлов, они все остаются внутри папки этой задачи.

## Процесс

1. Создать отдельную папку для задания.
2. Описать условие в `README.md`.
3. Положить решение в `src/`.
4. Добавить тесты в `tests/`, если они полезны для проверки поведения.
5. После завершения записать короткие выводы в `notes.md`.
